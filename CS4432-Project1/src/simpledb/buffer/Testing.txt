We initially attempted JUnit testing, but this proved nearly impossible due to the nature of simpleDB. As such, the majority of our
testing was done by running the SQL commands outlined in Examples.sql. As the buffer manager is an integral part of the system, any mistake
would (and did) lead to errors. 

The vast majority of our testing was performed by adding print statements at various points in the program.

Due to the vast size of most printouts, the complete outputs are not shown below. However, what is shown should be enough to prove the program works; the same
SQL commands could be run personally to see the full output.

----------------------------------
CLOCK POLICY:
The new toString() method prints out the current clock position. If the clock position is updated correctly and the SQL commands return the proper values,
the clock replacement policy is clearly functional. The BufferDisplay output was omitted for this test.
----------------------------------
